## 会社概要
- 会社名／屋号：合同会社サンプルスタジオ（屋号：Sample Studio）
- 代表者：山田 太郎
- 所在地：東京都渋谷区（自宅兼事務所）
- 形態：一人合同会社
- 並行運営：個人事業（同代表者によるフリーランス受託、屋号：Sample Works）

## 事業フェーズ
- プロダクト開発（自社 SaaS）と受託開発の二本柱で運営。受託で固定費を吸収し、SaaS で利益創出を目指す PMF 探索期
- 主な収益源：受託開発（売上 70%）／自社 SaaS サブスク（売上 30%）
- 財務状況の要点：累積損失あり（設立 2 期目）、月次キャッシュフローはプラス、半年分の運転資金あり

## プロダクト一覧

### TaskFlow（最優先）
- 概要：個人事業主向け請求書 + タスク管理 SaaS
- ターゲット：日本国内の一人法人・フリーランス（特に IT 系）
- 主機能：請求書発行・タスク管理・経費記録・freee 連携
- 進捗：β 版公開済み、有料プラン課金開始済み（MRR 約 30 万円）

### CodePost（保留）
- 概要：技術ブログ向けの執筆支援エディタ
- ターゲット：個人開発者
- 主機能：Markdown エディタ・SEO チェック・各種ブログ投稿連携
- 進捗：プロトタイプ段階、TaskFlow が落ち着くまで凍結

## 技術スタック
- フロントエンド：Next.js (App Router) + TypeScript + Tailwind CSS
- バックエンド：Next.js Route Handlers + Prisma
- DB：Supabase (PostgreSQL)
- 外部 API：Stripe（決済）、freee API（会計連携）、Anthropic API
- インフラ：Vercel（アプリ）、Supabase（DB / Auth）、Cloudflare（DNS / CDN）
- その他：受託案件で React Native / Python (FastAPI) / AWS にも触れる

## 行動・判断の必須ルール
1. 一人法人前提で考える。「人を雇う」前提の提案は避ける（必要なら業務委託・SaaS 化を優先）
2. 固定費最小化を優先する。月額サブスクの新規導入は ROI を必ず検証
3. キャッシュフロー優先。将来売上の楽観前提で設備投資・先行投資を判断しない
4. 法務・税務リスクには必ず注意喚起し、最終判断は弁護士・税理士に委ねる旨を付記
5. 回答は日本語で行う。コード内コメント・変数名は英語可
6. 複数選択肢があれば「一人で土日・夜間に実行できるもの」を優先提示
7. 受託案件の納期・品質を最優先しつつ、自社プロダクト開発時間を週 10 時間は確保する前提で助言
